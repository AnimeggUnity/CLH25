# 台科電 Modbus RTU 協議詳解

根據目前系統截獲的資料流，以下是 Master 指令與設備回傳訊息的完整拆解。

## 1. Master 發送指令 (Request)
雖然系統目前處於被動監聽模式，但在 Modbus 標準協議中，觸發該回傳封包的原始指令如下 (以 ID 01 為例)：

| 位元組 (Hex) | 名稱 | 說明 |
| :--- | :--- | :--- |
| `01` | 設備站號 | 指向 ID 01 電表 |
| `03` | 功能碼 | 讀取保持暫存器 (Read Holding Registers) |
| `00 00` | 起始位址 | 從暫存器 0x0000 開始讀取 |
| `00 18` | 讀取數量 | 讀取 24 個暫存器 (共 48 Bytes) |
| `45 C0` | CRC 校驗 | CRC-16 Modbus (Little Endian) |

**完整指令串：** `01 03 00 00 00 18 45 C0`

---

## 2. 設備回傳封包 (Response)
設備回傳的封包長度固定為 **53 Bytes**，結構拆解如下：

| 區段 | 長度 | 說明 |
| :--- | :--- | :--- |
| **ID** | 1 Byte | `01` ~ `06` |
| **Function Code** | 1 Byte | 固定為 `03` |
| **Byte Count** | 1 Byte | 資料內容長度，固定為 `30` (即 48 Bytes) |
| **Data Content** | 48 Bytes | 包含 24 個暫存器 (每個暫存器 2 Bytes) |
| **CRC16** | 2 Bytes | 校驗碼，用於驗證傳輸完整性 |

---

## 3. 資料內容分割與解析 (48 Bytes Payload)
這 48 Bytes 是我們最關心的核心資料，每 2 Bytes 代表一個暫存器 (`Reg`)：

| 暫存器編號 | 偏移量 (Hex) | 內容說明 | 解析邏輯 |
| :--- | :--- | :--- | :--- |
| **Reg 0** | `0-1` | 累積度數 Low Word | 與 Reg 1 組合成 32-bit kwh |
| **Reg 1** | `2-3` | 累積度數 High Word | [(Reg1 << 16) + Reg0](file:///c:/work/power/scan.py#220-302) / 100 = kWh |
| **Reg 2** | `4-5` | 電力特徵值 | 高位: 電壓; 低位: 電流/功率係數 |
| **Reg 3** | `6-7` | 狀態旗標 | `05A1`: 常開; `01D1`: 預付; `00D0`: 欠費; `00C0`: 預關 |
| **Reg 4 ~ 21** | `8-43` | 其他內部參數 | 包含頻率、功率因數或保留位 |
| **Reg 22** | `44-45` | 剩餘電量 Low Word | 與下一個暫存器組成 32-bit |
| **Reg 23** | `46-47` | 剩餘電量 High Word | [(Reg23 << 16) + Reg22](file:///c:/work/power/scan.py#220-302) / 100 = 餘額度數 |

### 實戰分割範例 (ID 01)：
原始傳輸：`01 03 30 AC1D 0003 F0CE 05A1`
## 控制指令集 (由嗅探錄製器分析得出)
除了讀取數據，我們也識別出了後台操作的指令規律：

### 1. 繼電器/模式控制 (Function 05: Write Single Coil)
| Coil 位址 | 意義 | 動作 (FF 00 / 00 00) |
| :--- | :--- | :--- |
| **`0002`** | 繼電器主開關 | `FF 00`: 開啟; `00 00`: 強制關閉 |
| **`000B`** | 預付模式開關 | `FF 00`: 進入預付模式; `00 00`: 切回普通模式 |
| **`0052`** | 預付手動控電 | `FF 00`: (預付中)送電; `00 00`: (預付中)斷電 |

### 2. 電量管理 (Function 10: Write Multiple Registers)
| Register 位址 | 長度 | 意義 | 換算公式 |
| :--- | :--- | :--- | :--- |
| **`0016`** | 32-bit (2 Regs) | 剩餘/儲值電量 | `Value / 100` = 度數 |

> [!IMPORTANT]
> **數據排列規律 (Word Swap)**:
> 儲值數據採用 32-bit 整數，排列順序為 **「低位暫存器 (Low Word) 在前，高位暫存器 (High Word) 在後」**。
> 例如儲值 **20.00 度** (2000 單位):
> - 2000 = `07 D0` (Hex)
> - 封包資料段: `07 D0` (Low) + `00 00` (High) = `07 D0 00 00`
4.  **取出 Reg 0/1**: `AC1D` (低位) + `0003` (高位) -> 十進制 `240669` -> **2406.69 kWh**
5.  **取出 Reg 3**: `05A1` -> 狀態為 **ON**
