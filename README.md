## 二、FC05 控制指令

```
TX: [ID] 05 [Addr H] [Addr L] [Val H] [Val L] [CRC]
```

### 線圈位址一覽

| 位址 | 值 | 功能 | 適用模式 |
|------|-----|------|---------|
| 0x000A | 0x0000 | 前置清除（用途未知，操作 0x000B 前必發） | 任意 |
| 0x000B | 0xFF00 | 進入儲值模式 | 非儲值 |
| 0x000B | 0x0000 | 退出儲值模式 | 儲值 |
| 0x0002 | 0xFF00 | 強制開電（繼電器 ON） | **非儲值模式專用** |
| 0x0002 | 0x0000 | 強制關電（繼電器 OFF） | **非儲值模式專用** |
| 0x0052 | 0xFF00 | 儲值模式開電（繼電器 ON） | **儲值模式專用** |
| 0x0052 | 0x0000 | 儲值模式關電（繼電器 OFF） | **儲值模式專用** |

> ⚠️ **0x0002 在儲值模式下無效**，繼電器控制必須改用 0x0052。

## 三、完整操作序列（由後台封包逆向分析）

以下序列皆由 RS485 旁聽抓取後台實際發送的封包歸納，每條指令間約 100ms。

### 強制開電
```
0x000A = 0x0000
0x000B = 0x0000
0x0002 = 0xFF00
```
效果：退出儲值模式（若在儲值模式），強制打開繼電器。

### 強制關電
```
0x000A = 0x0000
0x000B = 0x0000
0x0002 = 0x0000
```
效果：退出儲值模式（若在儲值模式），強制關閉繼電器。

### 切換儲值模式
```
0x000A = 0x0000
0x000B = 0xFF00
```
> ⚠️ 切換後繼電器狀態不確定，需另外發「儲值模式開電」才會供電。

### 退出儲值模式（不控制繼電器）
```
0x000A = 0x0000
0x000B = 0x0000
```
效果：僅退出儲值模式，不帶繼電器指令。退出後電表通常自動進入強制供電狀態。

### 儲值模式開電
```
0x0052 = 0xFF00
```

### 儲值模式關電
```
0x0052 = 0x0000
```

## 四、0x000A 的規律

- **每次操作 0x000B 之前，一定要先發 0x000A = 0x0000**
- 從未觀察到 0x000A = 0xFF00
- 具體功能未知，推測為模式切換前的解鎖或清除旗標
- 缺少此步驟可能導致模式切換失效或異常狀態

## 五、FC10 儲值寫入

```
TX: [ID] 10 00 16 00 02 04 [Low H] [Low L] [High H] [High L] [CRC]
```

| 項目 | 說明 |
|------|------|
| 位址 | 0x0016 |
| 長度 | 2 個暫存器（32-bit） |
| 排列 | Low Word 在前，High Word 在後 |
| 單位 | ×100（50.00 度 → 寫入 5000 = 0x1388） |
| **寫入值** | **新的總餘度（絕對值），非增量** |

**重要：後台的儲值邏輯是「讀取當前餘度 + 充值度數 = 新總量」，寫入的是新總量，不是要加多少。**

範例：當前 36.72 度，充值 40 度 → 寫入 76.72 度（7672 = 0x1DF8）

```python
def fc10_write_balance(ser, slave_id, kwh):
    units = int(round(kwh * 100))
    low_word  = units & 0xFFFF
    high_word = (units >> 16) & 0xFFFF
    payload = (bytes([slave_id, 0x10]) +
               struct.pack('>HH', 0x0016, 2) +
               bytes([4]) +
               struct.pack('>HH', low_word, high_word))
    pkt = payload + struct.pack('<H', crc16(payload))
    ser.write(pkt)
```

> ⚠️ 必須先在儲值模式（0x000B = ON）下才能有效寫入餘度。

## 七、FC03 讀取暫存器

```
TX: [ID] 03 00 00 00 18 [CRC]
RX: [ID] 03 30 [48 bytes] [CRC]
```

| Reg | 意義 | 換算 |
|-----|------|------|
| 0 | 累積度數 Low Word | (Reg1<<16 \| Reg0) / 100 = kWh |
| 1 | 累積度數 High Word | 同上 |
| 3 | 狀態旗標 | 見第六節 |
| 22 | 剩餘度數 Low Word | (Reg23<<16 \| Reg22) / 100 = 餘度 |
| 23 | 剩餘度數 High Word | 同上 |

---

## 八、FC10 修改裝置 ID

| 功能 | 位址 | 說明 |
|------|------|------|
| 修改裝置 ID | 0x0005 | 寫入新 ID（1~247），立即生效 |

> ⚠️ FC06 對 reg 0x0005 **無效**，必須用 FC10。
